Programação Estruturada

é um paradigma de programação com ênfase no uso de subrotinas, laços de repetição, condicionais e estruturas em bloco.

PS: Foi como programamos até agora aqui na disciplina

Programação Estruturada

- **Sequência**: Uma tarefa é executada após a outra, linearmente. - **Decisão**: A partir de um teste lógico, determinado trecho de código é executado, ou não. - **Iteração**: A partir de um teste lógico, determinado trecho de código é repetido por um número finito de vezes

Programação Estruturada

**Vantagens**: - É fácil de entender. - Ainda muito usada em cursos introdutórios de programação. - Execução mais rápida. **Desvantagens**: - Baixa reutilização de código - Códigos confusos: dados misturadis com comportamento

Programação Orientada a Objetos

Orientação a Objetos é um paradigma que considera que um programa formado por uma coleção de objetos que interagem entre si.

Objetivo: É trazer o mundo digital mais próximo do mundo real.

Programação Orientada a Objetos

Programação Orientada a Objetos

- Um programa formado por uma coleção de objetos que interagem entre si. - Os objetos conhecem muito bem a si mesmos - Respondem às mensagens de acordo com seus atributos (características) com seus próprios métodos (ações)
